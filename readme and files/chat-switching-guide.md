# Missing Frame - Шпаргалка для смены чатов

## 🔄 Когда менять чат

### Обязательные точки (10 смен):
1. **После Фазы 0** - Setup завершен
2. **После Фазы 1** - Auth готов
3. **После Фазы 2** - Dashboard готов
4. **После деплоя на Render** - Перед Фазой 3
5. **После Фазы 3** - Projects готовы
6. **После Фазы 4** - Tasks & Calendar готовы
7. **После миграции на Railway** - Перед Фазой 5
8. **После Фазы 5** - HR готов
9. **После Фазы 6** - Meetings/Risks готовы
10. **После Фазы 7** - Finance готов
11. **Финальный чат** - Фаза 8 (Polish)

### Дополнительные причины:
- Контекст чата > 100 сообщений
- Застрял на ошибке > 20 сообщений
- Переключаешься между модулями

## 💾 Что сохранять между чатами

### Обязательно сохрани:

1. **roadmap-v2-with-chat-switches.md** - главный план
2. **Текущий .env файл** (без секретов в чат!)
3. **Список завершенных задач** (чекбоксы из roadmap)
4. **URL хостинга** (если уже задеплоено)
5. **Последний commit SHA** (для ориентира)

### Как сохранять:

**Вариант 1: Локальная папка**
```
~/Documents/missing-frame-docs/
  - roadmap.md
  - env-example.txt
  - completed-tasks.txt
  - hosting-urls.txt
```

**Вариант 2: Notion/Obsidian**
- Создай страницу "Missing Frame Development Log"
- Обновляй после каждой фазы

**Вариант 3: GitHub Wiki**
- В репо создай Wiki
- Страница "Development Progress"

## 🚨 Чего НЕ делать

❌ **Не пиши просто "продолжаем"** - я не знаю что было
❌ **Не копируй весь предыдущий чат** - слишком много токенов
❌ **Не теряй roadmap** - это твоя карта
❌ **Не начинай новый чат в середине задачи** - доделай сначала

## ✅ Контрольный список перед сменой

Перед тем как начать новый чат:

- [ ] Завершил все задачи текущей фазы
- [ ] Сделал `git commit && git push`
- [ ] Протестировал функционал локально
- [ ] Если есть хостинг - проверил на production
- [ ] Обновил roadmap (отметил галочки)
- [ ] Сохранил важные файлы
- [ ] Записал что сделано и что дальше

**Только после этого** начинай новый чат.

## 🎯 Цель смены чатов

**Зачем это нужно:**

1. **Чистый контекст** - я быстрее понимаю что делать
2. **Меньше токенов** - экономия на API
3. **Фокус** - каждый чат = одна фаза
4. **История** - можно вернуться к старому чату и посмотреть как делали

**Аналогия:**
Смена чата = новая глава в книге. Каждая глава законченная, с четким началом и концом.
